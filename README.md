# Tg bot для распознования архитектруных памятников

План

## 1. Сбор данных

- Картинки: Собираем датасет фото каждого памятника с разных ракурсов (в том числе с um.mos.ru)
- Текст: Парсим um.mos.ru, citywalls.ru для получения детальной информации о каждом объекте
- Структура: Создаем БД где id_здания → [фото, название, описание, истории, архитектор]

## 2. Разработка ML модели

- Модель: Берем скорее всего предобученную CNN (ResNet, EfficientNet)
- detr network
- Задача: Дообучаем на своих данных для классификации архитектурных памятников
- Выход: Модель, которая по фото определяет id_здания

## 3. Бэкенд система

- Telegram Bot: Принимает фото, отдает красивый ответ
- ML API: Сервис, который прогоняет фото через модель
- Data Fetcher: Достает информацию из БД по предсказанному id + выдача ответа RAG (langchain)


## 4. Интеграция (опционально)

- Объединяем все компоненты в Docker-контейнеры
- Деплоим на облачный сервер
- Настраиваем логирование и мониторинг
